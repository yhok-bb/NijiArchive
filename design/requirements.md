## 1. ユーザーストーリー

ロードマップ Part 1 のユースケース（UC-1〜4）を分解したもの。IDは `US-<ペルソナ>-<連番>`。

### ファン視聴者（ペルソナA: みさき）

| ID | ストーリー | 対応FR |
|----|-----------|--------|
| US-F-1 | ファンとして、話題のライブアーカイブをトップページで見つけたい。見逃した配信を後から追体験するため | FR-1 |
| US-F-2 | ファンとして、回線状況が悪くても画質が自動で落ちて再生が止まらないでほしい。移動中でも視聴したいため | FR-1-3 |
| US-F-3 | ファンとして、動画の「盛り上がりポイント」にワンクリックでジャンプしたい。SNSで話題の瞬間だけ見たいため | FR-3-3 |
| US-F-4 | ファンとして、視聴中にコメントを投稿し、当時の盛り上がりを追体験したい | FR-3 |
| US-F-5 | ファンとして、前回の続きから再生を再開したい | FR-1-4 |
| US-F-6 | ファンとして、そのライブに関連するYouTube公式動画・切り抜きを一覧で辿りたい | FR-4 |

### エンジニア訪問者（ペルソナB: たかし）

| ID | ストーリー | 対応FR |
|----|-----------|--------|
| US-E-1 | エンジニアとして、視聴中の実際のビットレートとABR切り替えの発生をリアルタイムで見たい。「へえ、こうやって動いてるんだ」を体験するため | FR-6-1, 6-2 |
| US-E-2 | エンジニアとして、全視聴者のビットレート分布を見たい。実際のユーザーの回線環境を知るため | FR-6-4 |
| US-E-3 | エンジニアとして、ダッシュボードを閉じて普通の視聴体験にも戻りたい | FR-6-5 |

### 管理者（yhok）

| ID | ストーリー | 対応FR |
|----|-----------|--------|
| US-A-1 | 管理者として、MP4をアップロードするだけで複数解像度のHLSに自動変換・公開したい | FR-1-1, FR-7 |
| US-A-2 | 管理者として、変換の進捗と失敗を管理画面で把握したい | FR-7-1 |
| US-A-3 | 運営として、同一アカウントの同時視聴を1セッションに制限したい。アカウント共有を防ぐため | FR-2-2 |

## 2. 機能要件（FR）

Part 1 の FR に受け入れ条件と優先度を付与。優先度は Part 3 の「1〜2ヶ月で形にする場合の優先順位案」に従う。

| ID | 要件 | 受け入れ条件 | 優先度 | Step |
|----|------|-------------|--------|------|
| FR-1-1 | MP4アップロード→HLS自動変換 | アップロード完了後、人手の介在なく変換・R2配置まで完了する | Must | 1 |
| FR-1-2 | ブラウザでのHLS再生 | Chrome/Safari/Firefox最新版で再生できる | Must | 1 |
| FR-1-3 | ABR対応（1080p/720p/360p） | 帯域制限をかけると自動で低解像度に切り替わる | Must | 1 |
| FR-1-4 | 視聴履歴の記録 | 再訪時に前回位置から再開できる | Should | 2 |
| FR-2-1 | ユーザー登録・ログイン | email+パスワードで登録・ログイン・ログアウトできる | Should | 2 |
| FR-2-2 | 同時ログイン1セッション制限 | PC視聴中にスマホでログインすると後者がブロックされ、PC側は継続する | Should | 2 |
| FR-2-3 | 未ログイン時の視聴制限 | 未ログインでは視聴ページに到達できない <!-- 要確認: 「制限された範囲のみ視聴可」の選択肢もある。MVPは全面ログイン必須に倒した --> | Should | 2 |
| FR-3-1 | リアルタイムコメント投稿 | 投稿から他視聴者への表示まで1秒以内（NFR-1-2） | Should | 3 |
| FR-3-2 | コメントのタイムスタンプ紐付け | 再生位置に応じて過去コメントが流れる | Should | 3 |
| FR-3-3 | 盛り上がり自動検出 | 30秒窓のコメント急増が閾値超過でマークされ、シークバーに表示される | Could | 3.5 |
| FR-4-1〜3 | YouTube関連動画の集約 | 定期同期で一覧表示。動画本体は埋め込み/リンクのみ | Could | 7 |
| FR-5-1〜2 | 現在視聴者数の表示 | 5秒間隔で更新される | Should | 4 |
| FR-6-1 | 再生中ビットレート表示 | 現在の値がダッシュボードに出る | Must | 4.5 |
| FR-6-2 | ABR切り替えログ | 切り替え発生がタイムラインに追記される | Must | 4.5 |
| FR-6-3 | CDNキャッシュHIT/MISS表示 | 可能な範囲で表示（cf-cache-statusヘッダ利用） | Could | 4.5 |
| FR-6-4 | 視聴ビットレート分布の集計表示 | 複数ブラウザ同時視聴で分布が正しく集計される | Must | 4.5 |
| FR-6-5 | ダッシュボードのトグル | デフォルト非表示。表示/非表示を切り替えられる | Must | 4.5 |
| FR-7-1〜2 | 管理機能（動画CRUD・権限管理） | 管理者のみアクセス可能 | Must | 1〜2 |

## 3. 非機能要件（NFR）

| ID | 要件 | 目標値 | 検証方法 |
|----|------|--------|---------|
| NFR-1-1 | 再生開始レイテンシ | 3秒以内 | ブラウザDevToolsで計測 |
| NFR-1-2 | コメント反映 | 1秒以内 | 2ブラウザ間で実測 |
| NFR-1-3 | コメント負荷耐性 | 秒間100件で落ちない | ApacheBench |
| NFR-2-1 | 可用性 | 意図的ダウンタイム以外なし（24/365は非目標） | 稼働記録 |
| NFR-2-2 | エンコード失敗の隔離 | 変換失敗が視聴中ユーザーに影響しない | 失敗ジョブ注入テスト |
| NFR-3-1 | 認証情報の保護 | bcryptハッシュ化 | コードレビュー |
| NFR-3-2 | Presigned URL | 有効期限30分（初期値、実測で調整） | 期限切れURL直叩きで403 |
| NFR-3-3 | YouTube API規約遵守 | メタデータのみ・バッチ同期 | 設計レビュー |
| NFR-4-1〜2 | 可観測性 | OpenTelemetryトレーシング、Grafanaで可視化 | Step 5 完了基準 |
| NFR-4-3 | コードの可読性 | 第三者（採用担当者含む）が読める構造 | README・記事化 |
| NFR-5-1〜2 | コスト | 月数千円以内。R2エグレス無料を活用 | 月次コスト記録 |

## 4. 制約条件

- 開発者1人（yhok）、業務外の時間で開発
- 技術スタックは Rails 8 + Cloudflare エコシステム（学習目的との整合。→ ADR-0001, 0005）
- YouTube API利用規約・著作権法の遵守（→ ADR-0006）
- アーカイブ動画は自作または権利許諾済みの素材のみ。ニジガク公式映像の無断使用はしない

## 5. Out of Scope

マネタイズ / 決済 / 複数グループ横展開 / ネイティブアプリ。詳細は roadmap Part 1 §1-6。

## 6. 成功基準

事業指標ではなく学習・アピール指標で判断する（Part 1 §2-6）：技術記事5本以上 / 面接で設計判断を一貫説明できる / 第三者が読めるリポジトリ / コミュニティの反応 / 配信システムの腹落ち理解。
